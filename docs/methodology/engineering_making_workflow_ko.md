# 공학적 메이킹 워크플로우

## 목적

이 문서는 특정 도킹스테이션뿐 아니라 다른 설계도, CAD 모델, 하드웨어 제작 프로젝트에도 적용할 수 있는 공학적 메이킹 방법론입니다.

목표는 단순히 설계 파일을 만드는 것이 아닙니다. 목표는 사용자가 실제 재료와 도구로 완성할 수 있는 제작 시스템을 만드는 것입니다.

## 핵심 원칙

### 1. 설계는 형상이 아니라 사용 가능한 절차다

CAD 형상이 있어도 다음이 없으면 완성 가능한 설계가 아닙니다.

- 부품 이름
- 수량
- 재료
- 적층/가공 방향
- 조립 순서
- 접착/체결 위치
- 테스트 방법
- 실패 시 조정 방법

### 2. 재료가 설계를 지배한다

같은 형상도 재료가 바뀌면 좋은 설계가 아닐 수 있습니다.

예:

- 하드보드지: 넓은 접착면, 큰 탭, 적층 구조가 유리
- 아크릴: 레이저 커팅, 정확한 탭/슬롯, 취성 고려
- 합판: 결 방향, kerf, 탄성, 그을림 고려
- 3D 프린트: 오버행, 레이어 방향, 서포트, 수축 고려
- 금속판: 절곡 반경, 버, 체결구, 절곡 순서 고려

### 3. 전체 제작 전에 작은 실패를 먼저 만든다

전체 부품을 자르기 전에 테스트 쿠폰을 만든다.

테스트 쿠폰은 다음을 검증한다.

- 탭/슬롯 fit
- 실제 재료 두께
- kerf
- 구멍 크기
- 곡선 절단 가능성
- 접착 강도
- 부품 삽입/탈착 난이도

### 4. 초보자에게 어려운 것은 형상이 아니라 판단이다

초보자는 다음에서 주로 실패합니다.

- 어떤 부품인지 모름
- 몇 장 겹치는지 모름
- 어느 면에 풀을 바르는지 모름
- 어느 방향이 앞/뒤인지 모름
- 언제 fit을 확인해야 하는지 모름
- 실패했을 때 어떤 치수를 바꿔야 하는지 모름

따라서 좋은 설계는 판단을 줄이고 순서를 제공합니다.

## 전체 단계

```text
1. 목적 정의
2. 사용자와 사용 장면 정의
3. 재료/공법 확정
4. 실측 및 제원 수집
5. 기능 구조 설계
6. 제작 구조 설계
7. CAD/도면 작성
8. 테스트 쿠폰 제작
9. 1차 프로토타입 제작
10. fit/use 검증
11. 초보자용 문서화
12. 최종 제작 패키지 정리
```

## 1. 목적 정의

먼저 만들 물건의 목적을 한 문장으로 쓴다.

좋은 예:

```text
Xiaomi Watch S4를 한 손으로 올려놓으면 안정적으로 충전되는 하드보드지 도킹스테이션을 만든다.
```

나쁜 예:

```text
예쁜 도킹스테이션을 만든다.
```

목적 정의에는 다음이 들어가야 한다.

- 대상 물체
- 사용 행동
- 성공 상태
- 주요 제약

## 2. 사용자와 사용 장면 정의

다음 질문에 답한다.

- 누가 만드는가?
- 누가 사용하는가?
- 어느 정도 기술 수준인가?
- 어떤 도구가 있는가?
- 완성 후 어디에 놓고 쓰는가?
- 반복 사용인가, 1회성인가?

사용자가 초보자라면 설계 난이도보다 설명 난이도를 낮춰야 한다.

## 3. 재료/공법 확정

설계 전에 재료와 가공법을 확정한다.

필수 항목:

- 재료명
- 실제 두께
- 두께 편차
- 절단 방법
- 접착/체결 방법
- 허용 오차
- 표면 마감

예:

```text
재료: 하드보드지
두께: 1.5 mm
구조 두께: 2장 적층 = 3.0 mm
베이스: 3장 적층 = 4.5 mm
절단: 레이저 커터 또는 손칼
접착: 목공풀
```

## 4. 실측 및 제원 수집

카탈로그 수치보다 실제 측정값을 우선한다.

필수 실측:

- 대상 물체의 외형 치수
- 돌출부 치수
- 케이블/커넥터 치수
- 삽입/탈착에 필요한 여유
- 사용 중 움직이는 부품 치수
- 무게 또는 중심 이동

실측값과 가정값을 문서에서 분리한다.

## 5. 기능 구조 설계

물건이 해야 하는 기능을 쪼갠다.

예:

```text
충전기를 잡는다.
워치를 중심으로 안내한다.
스트랩을 통과시킨다.
케이블을 꺾이지 않게 뺀다.
책상 위에서 넘어지지 않는다.
초보자가 조립할 수 있다.
```

각 기능마다 물리 구조를 연결한다.

```text
충전기 고정 -> pocket + rear retainer
워치 안내 -> saddle + lower lip
케이블 정리 -> rear hole + channel + bend relief
안정성 -> wide base + layered base
조립성 -> tabs + glue faces + guide
```

## 6. 제작 구조 설계

제작 구조는 기능 구조와 다릅니다.

기능 구조가 “무엇을 해야 하는가”라면, 제작 구조는 “어떻게 만들 수 있는가”입니다.

검토할 항목:

- 부품 수가 너무 많지 않은가?
- 같은 부품을 여러 장 겹치는가?
- 좌우 대칭 부품이 헷갈리지 않는가?
- 접착면이 충분한가?
- 조립 중 각도를 잡을 방법이 있는가?
- 잘못 붙이면 되돌릴 수 있는가?
- 공구가 들어갈 공간이 있는가?

## 7. CAD/도면 작성

CAD 파일은 최소한 다음 모드를 갖추는 것이 좋다.

```text
assembly
fit_check
cut_layout_2d
cut_layout_preview
test_coupon_2d
novice_cut_layout_2d
individual_part_views
```

원칙:

- 3D 조립 미리보기와 2D 제작 도면을 분리한다.
- 시각 확인용 dummy와 실제 제작 geometry를 분리한다.
- 초보자용 단순 레이아웃을 별도 제공한다.
- 숫자 파라미터는 문서와 같은 이름을 사용한다.

## 8. 테스트 쿠폰 제작

테스트 쿠폰은 전체 제작 전 필수입니다.

포함할 것:

- 대표 슬롯
- 대표 탭
- 가장 중요한 구멍
- 가장 작은 곡선
- 케이블/부품 통과 샘플
- 각도 게이지 또는 alignment 샘플

판정 기준:

- 손으로 들어가는가?
- 너무 헐겁지 않은가?
- 접착 후 변형되는가?
- 실제 부품이 들어가는가?
- 칼/레이저로 깨끗이 잘리는가?

## 9. 1차 프로토타입 제작

처음부터 완성품을 만들려고 하지 않는다.

추천 순서:

1. 테스트 쿠폰
2. 핵심 fit 부품만 제작
3. 구조 프레임 제작
4. 전체 임시 조립
5. 실제 사용물과 fit 확인
6. 접착 고정

접착 전에는 반드시 실제 부품을 대본다.

## 10. Fit/Use 검증

검증은 “모양이 맞다”가 아니라 “사용된다”를 확인한다.

체크:

- 한 손으로 사용할 수 있는가?
- 삽입 방향이 자연스러운가?
- 케이블이 눌리지 않는가?
- 사용 중 흔들리지 않는가?
- 탈착이 쉬운가?
- 반복 사용해도 헐거워지지 않는가?
- 실패 시 조정할 수 있는가?

## 11. 초보자용 문서화

초보자용 문서에는 다음이 있어야 한다.

- 부품 번호
- 부품 수량
- 적층 수량
- 앞/뒤/위/아래 방향
- 접착 위치
- 조립 순서
- 건조 시간
- fit 확인 타이밍
- 실패 시 조정표

가능하면 “하지 말아야 할 것”도 적는다.

## 12. 최종 제작 패키지

완성 패키지에는 다음이 포함되어야 한다.

```text
source.scad
cut_layout.dxf
novice_cut_layout.dxf
test_coupon.dxf
assembly_guide.md
measured_specs.md
change_log.md
validation_report.md
```

## 품질 판단

좋은 설계의 기준:

- 목적에 맞다.
- 실제 재료로 만들 수 있다.
- 작은 테스트로 위험을 줄인다.
- 초보자가 부품을 구분할 수 있다.
- 접착/체결 위치가 명확하다.
- 실사용 검증 절차가 있다.
- 실패했을 때 조정할 파라미터가 명확하다.

## 공학적 결론

메이킹 프로젝트의 성공은 CAD 완성도가 아니라 제작 실패를 줄이는 정보 구조에서 결정됩니다.

좋은 워크플로우는 다음 순서를 지킵니다.

```text
목적 -> 사용자 -> 재료 -> 실측 -> 기능 -> 제작 구조 -> CAD -> 테스트 쿠폰 -> 프로토타입 -> 검증 -> 초보자 문서화
```

이 순서를 지키면 다른 설계도에도 같은 방식으로 적용할 수 있습니다.
