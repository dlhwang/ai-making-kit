# Making Requirements Analysis

Purpose: clarify what the physical project must accomplish.

## Depth Selection

Use minimal depth when the object is simple, low risk, and personally used.

Use standard depth for reusable designs, novice builders, multiple materials, or meaningful fit constraints.

Use comprehensive depth for safety-critical, load-bearing, electrical, heat, public release, or paid fabrication work.

## Analyze

- Purpose
- Target user
- Builder skill level
- Use environment
- Primary use sequence
- Required functions
- Fit requirements
- Build constraints
- Tools available
- Deliverables needed
- Success criteria
- Scope out
- Risks
- Open questions

## Output

Create `making-docs/inception/requirements/making-requirements.md`.

Include:

- Purpose
- Scope
- Users/builders
- Required functions
- Constraints
- Deliverables
- Acceptance criteria
- Risks
- Open questions
