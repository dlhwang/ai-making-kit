# Communication and Handoff Design

Purpose: make sure the deliverable can be understood, approved, adopted, or operated by the people who receive it.

Execute this stage when the work:

- Requires approval.
- Changes a process or responsibility.
- Goes to executives, customers, partners, auditors, or another team.
- Needs follow-up actions after delivery.
- Will be reused by people who were not involved in creating it.

## Design

Define:

- Primary audience
- Secondary audiences
- Core message
- Required context
- Delivery channel
- Timing
- Owner
- Reviewer or approver
- Handoff package
- Follow-up expectations
- Escalation path

## Output

Create `blc-docs/execution/communication-and-handoff-design.md`.

Include:

- Audience map
- Message summary
- Handoff checklist
- Follow-up action table
- Risks or sensitivities
