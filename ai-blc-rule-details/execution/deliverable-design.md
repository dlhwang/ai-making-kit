# Deliverable Design

Purpose: design the deliverable before drafting it.

## Design

Define:

- Audience
- Purpose
- Format
- Outline or structure
- Required evidence
- Claims and assumptions
- Tone
- Review path
- Acceptance criteria

## Output

Create `blc-docs/execution/deliverable-design.md`.

For low-risk work, this can be a short section inside the execution plan. For high-impact work, wait for approval before drafting.
