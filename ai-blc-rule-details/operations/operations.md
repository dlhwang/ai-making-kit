# Operations

Purpose: ensure the work is used, followed up, and improved.

Execute this stage when the deliverable creates ongoing actions, process changes, recurring reporting, stakeholder adoption, or measurable outcomes.

## Define

- Follow-up actions
- Owners
- Due dates
- Metrics
- Review cadence
- Communication rhythm
- Escalation path
- Retrospective date
- Improvement backlog

## Output

Create `blc-docs/operations/operations-plan.md`.

Update `blc-docs/blc-state.md` with:

- Completed deliverables
- Follow-up actions
- Next review date
- Open risks
- Workflow status
