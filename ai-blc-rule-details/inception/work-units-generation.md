# Work Units Generation

Purpose: break larger work into executable units.

Execute this stage when the work has multiple deliverables, owners, dependencies, milestones, or review gates.

## Unit Fields

Each unit should include:

- Unit name
- Purpose
- Inputs
- Outputs
- Owner or role
- Dependencies
- Review gate
- Done criteria
- Risks

## Output

Create `blc-docs/inception/work-units/work-units.md`.

Wait for explicit approval.
