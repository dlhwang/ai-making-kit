# Work Planning

Purpose: decide how the work should proceed.

## Inputs

Use approved artifacts from:

- Work Context Detection
- Current-State Analysis, if executed
- Work Requirements Analysis
- Stakeholder Scenarios, if executed

## Plan

Define:

- Deliverables
- Sequence of work
- Needed stages
- Owners or roles
- Dependencies
- Review gates
- Timeline
- Risks and mitigations
- Definition of done

## Output

Create `blc-docs/inception/plans/execution-plan.md`.

Include:

- Recommended workflow
- Stage list with execute/skip decisions
- Work breakdown
- Review and approval gates
- Timeline or sequencing
- Risks
- Next action

Wait for explicit approval before execution.
