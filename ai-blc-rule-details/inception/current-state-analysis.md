# Current-State Analysis

Purpose: understand how the work is currently done before proposing changes.

Execute this stage when the work affects an existing process, report, policy, team routine, customer journey, operating model, or decision flow.

## Analyze

- Current process or artifact
- Current owners and participants
- Inputs and outputs
- Tools and systems used
- Approval path
- Pain points and bottlenecks
- Known risks
- What must not change

## Output

Create `blc-docs/inception/current-state/current-state-analysis.md`.

Include:

- Summary
- Current workflow
- Stakeholder map
- Pain points
- Constraints
- Risks
- Opportunities
- Assumptions and open questions

Wait for explicit approval before using this analysis as a planning baseline.
