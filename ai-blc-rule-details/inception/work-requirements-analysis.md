# Work Requirements Analysis

Purpose: clarify what the work must accomplish.

## Depth Selection

Use minimal depth when the request is clear, low risk, and has a small audience.

Use standard depth for normal cross-functional work, reusable deliverables, or work with meaningful review.

Use comprehensive depth for executive, legal, financial, compliance, customer-facing, or high-impact work.

## Analyze

- Business purpose
- Problem or opportunity
- Expected deliverables
- Stakeholders
- Scope in
- Scope out
- Success criteria
- Constraints
- Source materials
- Risks
- Open questions

## Questions

If required information is missing, create `blc-docs/inception/requirements/requirements-questions.md`.

## Output

Create `blc-docs/inception/requirements/work-requirements.md`.

Include:

- Purpose
- Scope
- Deliverables
- Stakeholders
- Acceptance criteria
- Constraints
- Risks
- Open questions
- Recommended approval statement

Wait for explicit approval.
