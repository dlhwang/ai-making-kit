# AI-BLC Process Overview

AI-BLC has three phases:

1. Inception: understand the work.
2. Execution: create and verify the deliverables.
3. Operations: make the result usable and sustainable.

## Adaptive Stage Selection

Always execute:

- Work Context Detection
- Work Requirements Analysis
- Work Planning
- Deliverable Creation, if a deliverable is requested
- Review and Validation, if a deliverable is created

Execute conditionally:

- Current-State Analysis, when existing work must be understood first
- Stakeholder Scenarios, when multiple people or teams are affected
- Work Structure Design, when a new process, policy, operating model, or reporting structure is needed
- Work Units Generation, when the work has multiple owners, dependencies, or milestones
- Quality Criteria, when accuracy, compliance, confidentiality, executive review, or external use matters
- Operations, when follow-up, adoption, metrics, or repeated execution matters

## Text Workflow

```text
User Request
  -> Work Context Detection
  -> Current-State Analysis, if needed
  -> Work Requirements Analysis
  -> Stakeholder Scenarios, if needed
  -> Work Planning
  -> Work Structure Design, if needed
  -> Work Units Generation, if needed
  -> Deliverable Design, if needed
  -> Quality Criteria, if needed
  -> Deliverable Creation
  -> Review and Validation
  -> Operations, if needed
  -> Complete
```

## Human Control Points

Wait for explicit approval after:

- Current-State Analysis
- Work Requirements Analysis
- Stakeholder Scenarios
- Work Planning
- Work Structure Design
- Work Units Generation
- High-impact Deliverable Design
- Review and Validation

## Guiding Principle

The AI should make the next useful step easier, not hide judgment inside finished-looking output.
